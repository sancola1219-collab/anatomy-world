# lesson.md — 生物構造教學室：設計決策與工作筆記

> 摘要：一個離線可用的互動教學網頁，依分類學呈現動物／植物／真菌的寫實解剖 SVG，支援遠看整體與近看器官；本檔記錄關鍵決策、修正與確認，以及「為什麼重要」，供日後接手。

## 架構決策（Confirmed）

### 1. 純前端、零依賴、離線可用
- **做法**：`index.html` + `css/style.css` + `js/app.js` + `data/*.js`，用瀏覽器直接開，無需伺服器、無 npm、無外部 CDN。
- **為什麼重要**：教學現場網路不一定穩；零依賴代表五年後仍能開，也讓日後任何模型（Codex / Claude Code）都能無環境負擔地改。

### 2. 資料與引擎分離（外掛式註冊表）
- **做法**：引擎 `app.js` 不知道任何物種；每個 `data/*.js` 呼叫 `window.ANATOMY.register({...})` 把物種掛進去。分類骨架（界／類群）集中在 `app.js` 的 `TAXONOMY`。
- **為什麼重要**：要「包含所有種類」內容量極大。分離後可平行產內容、單檔擴充、單檔除錯，壞一個檔不會拖垮整個 App。這是日後擴充的主要施力點。

### 3. 「遠看／近看」用兩種機制疊加
- **做法**：(a) 自由縮放平移（滾輪 + 拖曳，CSS transform）；(b) 結構化「視圖分頁」逐系統拆解（外部形態 / 骨骼 / 內臟…）。
- **為什麼重要**：純自由縮放無法表達「不同系統」；純分頁又不夠直觀。兩者疊加＝既能無段放大看細節，又能有邏輯地切換構造層次，符合教學動線。

### 4. 寫實風格用「著色 SVG 解剖圖」而非照片
- **做法**：SVG + 漸層明暗；標註引線包在 `<g class="labels">` 可切換；主要器官加 `data-part` 可點擊高亮。
- **為什麼重要**：離線環境無法內嵌大量授權照片；教科書式著色解剖圖反而標註更清楚、可縮放不失真、檔案小。`data-part` 與 `parts[]` 名稱一致是互動高亮的關鍵約定，改圖時務必同步。

## 修正記錄（Corrections）

### C1. `data-part` 無對應 `parts[]` 項目（鯊魚「身體」、蕨類「羽片」）
- **現象**：整合驗證發現這兩個 SVG 元素有 `data-part` 但同視圖 `parts[]` 沒列。
- **修正**：在對應視圖 `parts[]` 各補一項同名條目。
- **為什麼重要**：`data-part` 名稱與 `parts[].name` 必須一致，才能「點器官↔點清單」雙向高亮。不一致時點器官仍會閃（引擎直接用 dataset 查），但該部位不會出現在該視圖的清單，教學上會漏掉。這是新增/改圖最容易犯的錯，列為除錯第 3 條。

## 驗證結果（Confirmed，2026-07-06）
用 Node 模擬瀏覽器 + 實機 preview 全部通過：
- **45 物種、84 視圖、517 標註**；3 界 19 類群全部上樹。
- `node --check` 9 檔全過；DOMParser 解析 84 視圖 **0 個 parsererror**；瀏覽器 console **0 錯誤**。
- id 衝突 0、分類歸位錯誤 0、`data-part`↔`parts` 不匹配 0、缺標註層 0、SVG 尖括號不平衡 0。
- 互動實測：切物種/切視圖/縮放(100%→156%→重設)/跨物種搜尋器官（搜「心臟」命中 9 物種）/標註開關 皆正常。
- **為什麼重要**：這套「Node 語法檢查 + DOMParser 渲染檢查 + preview 互動檢查」三層驗證，是本專案沒有測試框架下的標準驗收流程，接手者改完照跑即可（見 HANDOFF.md 第 5 節）。

### 已知環境限制
- preview 的**截圖工具**對大量帶 `drop-shadow` 濾鏡的 SVG 會逾時，但頁面本身反應正常（eval 皆秒回）。驗證改用 DOM 檢查與 DOMParser，不靠截圖。日後若要截圖，可暫時移除 `#svgHost svg` 的 `filter` 再截。

## 已知約定（給接手者的重點）
- 新增物種：複製既有 `register({...})` 區塊，改 `id`（全域唯一）、`kingdom`、`group`（須為 `app.js` `TAXONOMY` 內的 key）。
- SVG 只寫內層標記，引擎自動包 `<svg viewBox>`。
- 漸層 id 要加物種前綴避免全頁衝突。
- `data-part` 名稱必須等於同視圖 `parts[].name`，否則點擊高亮失效。
- 詳細規格見 `data/SCHEMA.md`；接手流程見 `HANDOFF.md`。
