## 0. 這是什麼
一個**離線互動生物構造教學網頁**。依分類學（界→類群→物種）呈現動物、植物、真菌的**寫實著色解剖 SVG**，使用者可**遠看整體、近看器官**（滾輪縮放＋拖曳平移），並用分頁**逐系統拆解**（外部形態／骨骼／內臟…）。純前端、零依賴。

**啟動方式**：用瀏覽器直接打開 `index.html`。沒有建置步驟、沒有伺服器、沒有 npm。

## 2. 執行流程（邏輯走一遍）
1. `index.html` 依序載入 `app.js`（定義 `window.ANATOMY`）→ 各 `data/*.js`（每個物種呼叫 `ANATOMY.register(org)` 塞進註冊表）→ 最後 `<script>ANATOMY.boot()</script>`。
2. `boot()` → `App.init()`：
   - `buildTree()`：依 `TAXONOMY`（界→類群）＋註冊表，產生左側可折疊分類樹。空的類群不顯示。
   - `bindViewer()`：綁定滾輪縮放、拖曳平移、工具列（放大/縮小/重設/全螢幕）、標註開關。
   - `bindChrome()`：搜尋（過濾樹）、主題切換、說明視窗。
   - 預設載入註冊表第一個物種。
3. 點物種 → `load(id)`：
   - `renderViewTabs()`：依 `org.views` 產生上方分頁。
   - `renderView()`：把目前視圖的 `svg` 包進 `<svg viewBox>` 注入畫布；重設縮放；綁定 `[data-part]` 熱點點擊。
   - `renderInfo()`：右側顯示名稱/學名/分類 chip/摘要/部位清單。
4. 互動：
   - 滾輪／工具列 → `zoomAt()` 改 `view.scale/x/y` → `apply()` 套 CSS transform。
   - 點器官或點右側部位清單 → `flashPart(name)`：在 SVG 找 `[data-part="name"]` 閃爍高亮；若該部位在別的視圖，自動切過去再高亮。
   - 標註開關 → `svgHost` 加/移除 `.hide-labels`，CSS 隱藏 `.labels`。

**關鍵資料流**：`data/*.js` 的 `register()` 是唯一的內容來源；`app.js` 完全通用、不寫死任何物種。

## 3. 資料契約（改內容只需懂這個）
每個物種 = 一次 `window.ANATOMY.register({...})`。完整欄位見 `data/SCHEMA.md`。最重要的四條約定：
1. **SVG 只寫內層標記**，不要外層 `<svg>`（引擎會用 `view.viewBox` 自動包）。
2. **標註**（引線＋文字）包在 `<g class="labels">…</g>` 內 → 才能被「標註開關」控制。
3. **可點器官**加 `data-part="部位名"`，且名稱**必須等於**同視圖 `parts[].name` → 才能點擊/清單高亮。
4. **漸層等 id 全檔唯一**，加物種前綴（如 `human-muscle`）→ 避免多物種同頁衝突。
5. `group` 必須是 `app.js` `TAXONOMY` 裡存在的 key；否則該物種不會出現在樹上。
6. **動畫是自動的**（`js/anim.js`）：`data-part` 只要用慣用器官名就自動有生理動畫 — 心臟(心)→跳動、肺/鰓→呼吸、血/動脈/靜脈→脈動、翅/翼/鰭/尾/觸手/觸角/腕/鞭毛/纖毛→擺動。**過程視圖**（`view.name/id` 含 生命週期/變態/世代交替/授粉/受精/萌發/出芽）會改成逐階段循環播放。不必在 data 檔寫任何動畫。要改對應規則就編 `js/anim.js` 的 `classify()` 與 `PROCESS` regex。

## 4. 常見任務怎麼做（給接手模型的操作手冊）

### A. 新增一個物種
1. 讀 `data/SCHEMA.md`。
2. 在對應 `data/*.js` 複製一個既有 `register({...})` 區塊。
3. 改 `id`（全域唯一）、`name`、`latin`、`emoji`、`kingdom`、`group`、`summary`。
4. 畫 `views[]`：每個視圖給 `viewBox`、內層 `svg`、`parts[]`。遵守第 3 節四條約定。
5. 存檔，重新整理瀏覽器即可看到。

### B. 新增一個分類類群（例如新增一個「綱」）
- 到 `js/app.js` 的 `TAXONOMY` 物件，在對應界的 `groups` 加一個 key（含 `name`、`emoji`）。

### C. 修圖 / 加標註
- 直接編對應視圖的 `svg` 字串。加標註記得放進 `<g class="labels">`；要可點就加 `data-part` 並在 `parts[]` 補同名項目。

### D. 除錯清單（東西沒出現時依序檢查）
1. 物種沒出現在樹上 → `group`/`kingdom` key 拼錯，或 `register` 沒被呼叫（該 data 檔語法錯誤導致整檔中止）。開瀏覽器 DevTools Console 看紅字。
2. 圖破圖 → 該視圖 SVG 樣板字串沒閉合、標籤沒關、或 `&`/`<` 未跳脫。
3. 點器官沒反應 → `data-part` 與 `parts[].name` 不一致。
4. 兩個物種互相干擾（顏色跑掉）→ 漸層 id 撞名，加前綴。
5. 標註關不掉 → 標註沒包在 `<g class="labels">`。

## 5. 驗證方式（改完怎麼確認沒壞）— 三層驗證
1. **語法層**：`node --check data/xxx.js`（或一次全部：`for f in data/*.js; do node --check "$f"; done`）。
2. **整合層**：`node verify.js`（專案根目錄，零依賴）。它模擬瀏覽器載入全部檔案，檢查：註冊總數、id 衝突、group/kingdom 合法性、`data-part`↔`parts[].name` 一致性、缺標註層、SVG 尖括號平衡，並印出各界/類群分佈。**全部應為 0 錯誤**。
3. **互動層**：瀏覽器開 `index.html`（或 `python -m http.server` 後開）：DevTools Console 無紅字 → 任選物種 → 切每個視圖都顯示 → 滾輪縮放 → 點器官會高亮 → 標註開關有效 → 搜尋器官名（如「心臟」）找得到跨物種結果。

> `verify.js` 是本專案的核心驗收工具，改完內容先跑它。詳細逐項結果與已知限制記在 `lesson.md`。

## 6. 後續工作與記憶（接手流程）
- **待辦 / 可擴充方向**：見本檔第 7 節與 `lesson.md`。
- **專案記憶**：`memory/` 內有跨對話筆記；主要決策的「為什麼」在 `lesson.md`。接手時**先讀 `lesson.md` → 本檔 → `data/SCHEMA.md`**，即可安全開工。
- **給 Codex 的提示**：這是純靜態前端，無測試框架；驗證靠「node --check 語法 + 瀏覽器目視」。不要引入建置工具或框架（會破壞「離線零依賴」這個核心約束）。
- **給 Claude Code 的提示**：擴充內容時用平行子代理，一個 data 檔一個代理，嚴格遵守 `data/SCHEMA.md`，最後由主代理整合並逐項跑第 5 節驗證。
